# Comet(彗记)v0.0.5 发布说明 —— 蜕变

**发布日期:** 2026 年 6 月 27 日 ｜ **版本代号:** 蜕变(TuiBian · The Agent that Evolves)

Comet v0.0.4「行者」让 Agent 学会了「自己干活」——一句话需求 → 自主深度研究 → 结构化带来源报告 → 定时自动跑 → 主动推送手机。但「**跑得动**」不等于「**跑得准**」:报告没人复核、引用错位无人发现、token 月底账单只见总数不知去向、记忆萃取出错用户也无路径告诉系统「这条记错了」。**v0.0.5「蜕变」的主线就是让 Agent 完成一次完整的迭代进化——从「一次性跑通」升级为「自我量化 + 自我审视 + 自我修正 + 人类反馈」的完整闭环**。我们一次性补齐了进化所需的四块拼图:**评测体系**(自制中文集 + 业界公认基准 C-MTEB / HotpotQA 对标,**自我量化**——知道自己处于什么水平)、**Verifier Loop 自闭环质量保障**(独立 LLM-as-judge 按 6 维 Rubric 评分 + 智能修复策略回炉,**自我审视 + 自我修正**)、**全链路可观测**(基于 OpenTelemetry GenAI 规范的 trace / span + 精确到元的成本核算,**知道自己是怎么做到的**)、**记忆审查与人类反馈数据池**(用户审查 + 纠错沉淀给下一阶段 self-improvement loop 训练信号,**接入人类反馈**)。当 Comet 不仅会跑、还能用业界标准指标量化自己、能让独立审稿人审视并修正自己、能下钻每一步审视自己的执行链路、还能听见用户的反馈反过来提升自己——它就完成了一次真正意义上的「**蜕变**」:从「会做事的 Agent」进化为「**会进化的 Agent**」。

## 🦋 一、核心升级概览

### 1. 评测体系 v1 + 公共基准对标 📊 ——「自我量化」

- **自建中文 gold 集**(`api/eval/`):覆盖 RAG 检索 / 记忆图谱召回 / 三元组抽取 / 实体去重四个维度,用 Recall@k / MRR / nDCG / P/R/F1 等业界标准指标量化效果——**RAG 混合检索 Recall@5 = 1.0 / nDCG@5 ≈ 0.99**、**记忆图谱检索 Recall@5 ≈ 0.82 / MRR ≈ 0.73**、**三元组抽取实体级 F1 ≈ 0.82**、**实体去重 Precision = 1.0**(零误合并)。
- **架构独立**:`api/eval/` 与生产代码零依赖耦合(独立 `EVAL_USER_ID` / 独立 `.env.eval` / 独立 session),不污染生产数据、可重入、跑前自检 + 双输出(指标表 + 逐条明细),便于定位问题。**数据驱动迭代真实样例**:评测明细发现萃取把「复旦大学」错抽成「用户的大学」→ 定位到提示词缺专名优先规则 → 改完记忆 Recall@5 **从 0.55 提到 0.82**。
- **公共评测基准对标**(三层评测结构):
  - **L1 自制中文集** → 证明产品自洽
  - **L2 C-MTEB T2Retrieval**(中文检索业界基线)→ 证明通用检索能力:**混合检索 nDCG@10 = 0.9818**(vs 纯向量 0.9699 +1.2pt,vs BM25 0.881 +10pt);**MRR@10 = 1.0**;意外发现:rerank 在小语料下(300 篇)反而轻微伤害 0.9818→0.9777,生产大语料才反转——为「rerank 何时启用」给出数据依据。
  - **L3 HotpotQA distractor**(多跳推理基线)→ 证明端到端 RAG + Verifier 设计:**200 题 EM = 62% / F1 = 74.49% / Recall@4 = 95.75%**;每题独立 user_id 灌入 → 混合检索 top-4 → 答多跳题 → 立刻清,题间零干扰。
- **诚实声明**:HotpotQA 2018 年发布主流 LLM 可能见过,只作系统设计对比、不作绝对水平断言。**「不卖做不到的事」**:原计划的 LongMemEval(英文长对话记忆基准)实测 oracle 2 题全 0(中文优先项目硬上数据难看),工程上明确**砍掉而非硬上**——能砍掉的判断比加上去更难。

### 2. Verifier Loop · Agent 自闭环质量保障 🔁 ——「自我审视 + 自我修正」

- **范式**:落地 **Loop Engineering**(2026.6 Anthropic 提出)——Agent 从「LLM 自循环」升级为「**外部系统驱动的循环 + 独立验证 + 智能修复 + 状态持久化**」,三个解耦原则:Verify ⊥ Generate / Controller ⊥ Task / State ⊥ Process。
- **6 维 Rubric 直接对齐 ① RAGAS**:覆盖度 / 引用对齐 / 论证深度 / 时效性 / 相关性 / 可读,每维 0~5 分加权,总分 ≥0.75 + 全部硬门槛过 = 通过。**评测-生产一致性**——离线发现的弱项就是生产防的弱项。
- **双 Verifier + 工厂降级**:`SameModelVerifier`(同模型自评,基线)+ `CrossModelVerifier`(跨家族,如 generator 用 DeepSeek、verifier 用 GLM)+ `build_verifier` 工厂(配 cross 但未配 verifier 模型时自动降级到 same 并 warning)。
- **A/B 实验证明跨家族必要性**(HotpotQA 20 题/组):
  - same(self-critique):judge 通过率 **0%** / EM 一致率 40%
  - **cross**(deepseek + glm-4-flash):judge 通过率 **95%** / EM 一致率 **65%**
  - 结论:self-critique 完全不可用,**跨家族独立审稿才是数据驱动的工程必选**。
- **智能 Repair 策略**(Policy 决策树):按问题类型自动选最经济的修复——
  - `PatchRepair` 贪心补丁(从 missing_coverage / wrong_citations 抽子查询补搜)→ coverage / faithfulness / timeliness 不达标
  - `ChapterRewrite` 章节重写(与 artifact.headings 求交集**防 verifier 编造章节名**)→ depth / relevance 不达标
  - `ForceExceed` 兜底停(沿用最后一次 artifact 返回)→ 超限或 ≥3 维全面烂
- **状态外置**:`loop_runs` + `loop_iterations` 两表,worker 重启可恢复;每轮 verify 完立即写库,有完整 audit trail。
- **接入点 wire-up 只多一行**:深度研究 `engine.py` 末尾 + 定时任务 `_check_loop_passed` 反查 LoopRun 不合格不推送 + HotpotQA A/B runner——**直接证明 LoopController 抽象成立**。
- **可视化**:研究报告页加 `QualityCard` 雷达图(6 维评分对比硬门槛虚线)+ 各轮 feedback Collapse;HomePage 加「Loop 健康度」卡(总运行 / 一次通过率 / 平均迭代 / 平均评分 + 失败维度归因 Top + verifier_kinds 分布);定时任务运行历史每条 run 加 verified 徽章。

### 3. 全链路可观测 + Token 成本核算 🔍 ——「知道自己是怎么做到的」

- **范式**:遵循 **OpenTelemetry GenAI semantic convention**(`gen_ai.system / gen_ai.request.model / gen_ai.usage.input_tokens` 等标准属性),数据落自有 PG 表 `agent_traces` + `agent_spans`——**未来导出到 OTel Collector / Jaeger / Tempo / LangSmith 改输出格式即可,零迁移成本**。
- **全链路埋点 7 处**:
  - `LLMClient.chat/embed/vision/rerank` 自带 `llm_call` span 记 model + tokens + cost + 600 字 request/response 预览
  - 深度研究 7 阶段全包 span(规划 / 检索带子 tool_call / 逐源提炼 / 反思 / 大纲 / 写章节 per N / 汇总)+ loop_run_id 关联
  - 对话编排双路径(function calling + ReAct)包 llm_span,工具调用包 `tool_call` span
  - Loop Controller 的 verifier + repair 包 span,`IterationOutcome.id` 预生成 UUID 与 LoopStore 共用 → **iteration_id 精确绑定**,Verifier Loop 黑盒被打开
  - 对话主路径 `chat_service._run_chat_turn_bg` 包 `trace`,trace_id 写进 `message.meta_data`
  - LangChain `ChatOpenAI(stream_usage=True)` 让流式响应也带 usage_metadata
  - 子模块用 `push_llm_usage(resp, model)` 把 usage 推到外层 span,**避免嵌套重复计费**
- **精确成本核算**:`pricing.py` 维护 20+ 常用 OpenAI 兼容 model 单价表(智谱 / DeepSeek / 通义 / 豆包 / OpenAI 等),每次 LLM 调用算 `cost_cny`,trace 级汇总 `total_cost_cny` + `total_tokens`;`COMET_PRICING_OVERRIDES` 支持 .env 覆盖。
- **异步落库**:asyncio.Queue 批量异步写库(默认 50 条 / 1s),业务主流程**不阻塞**;队列满兜底丢最早 span 并 warning,可用性优先。
- **前端**:
  - 独立 `/traces` 列表页(KPI 卡 + Segmented 筛选 + 紧凑表格行 grid 对齐 + 状态点圆形指示)
  - `TraceTimeline` Gantt 时间线(每 span 一条横条按耗时缩放,颜色按 type,内联手风琴展开)+ `TraceNarrative` 流程解读(按 span 嵌套层级拆解 Agent 决策过程,**用户视角中文化**,LLM/MCP/tokens 等专业英文保留)
  - **反向入口**:研究报告页「执行轨迹」按钮 → `/traces?task_id=...` 自动展开;定时任务运行历史行「轨迹」入口;**每条 AI 对话消息底部加「执行轨迹」按钮**(基于 `message.meta_data.trace_id`)
  - HomePage 加 `CostCard`:4 宫 KPI(总 trace 数 / 总 tokens / 总成本 CNY / 平均每次任务成本)+ 按 model 成本饼图 + 按 task_type 横向柱图,Segmented 切 7 / 30 / 90 天
  - 手机端 Drawer 改 `placement="bottom"` 修友好

### 4. 记忆审查 + 人类反馈数据池 🧑‍⚖️ ——「接入人类反馈,形成进化闭环」

- **痛点**:V0.0.5 之前 Comet 萃取出错的实体(如把「我妹妹林晓」萃成「林晓」「妹妹」两个独立实体),用户**没路径告诉系统「这条记得错了」**。
- **设计目标**:让 V0.0.5 的进化闭环接入「**人类反馈**这一公里」——任务内自修正(② Verifier Loop)+ 跨任务人类反馈(⑤),进化叙事才完整。
- **新增数据池表**:`memory_corrections`(action[confirm/correct/delete] + entity_id + before/after 快照 JSONB + reason),既保留**操作历史**(可审计、可回滚)又有**结构化反馈数据**(给 V0.0.6 self-improvement loop 消费);每次写操作走「**before 快照 → 写 Neo4j → after 快照 → 落 PG**」四步,审计完整。
- **Neo4j Entity 加 `human_verified` 动态属性**(无 schema 变更),回写图谱后**主动召回时给最高权重**,形成「**人类反馈 → 记忆质量提升**」的**实时闭环**。
- **前端 ReviewPanel(MemoryPage 第 5 Tab)**:
  - **Tab 1 我的记忆全景**:4 宫 KPI(已记住 / 长期 / 已确认 / 待确认,后者可点击直接跳 Tab 2)+ 类型饼图 + 置信度直方按区间红橙黄绿配色可点跳 + 30 天新增趋势渐变填充 + 纠错统计 Tag
  - **Tab 2 审查纠错**:Segmented 筛选栏(置信度 ≤0.6/≤0.75/≤0.9/全部 + 仅未确认/含已确认 + 类型 Select)+ 实体卡片(左侧色条 + 百分比胶囊 + 类型/长期/已确认 Tag + 三按钮 确认/修正/删除)
  - 手机端响应式:三按钮 grid 1fr 等分撑满,筛选 Segmented block=true
- **明确不做**:自动消费 corrections 反馈训练(V0.0.6 self-improvement loop 的事)、暴露技术指标到 UI(普通用户违和)、离线评测结果可视化 Tab(开发者本地工具)。

## 🧭 未来展望

v0.0.5「蜕变」给 Agent 装上了完整的进化闭环——**自我量化**(评测体系 + 公共基准)、**自我审视**(Verifier Loop 独立审稿)、**自我修正**(Patch / 章节重写智能 Repair)、**人类反馈**(记忆审查数据池)、**全程可观测**(OTel Tracing + 成本核算)。Agent 第一次拥有了「不只是会做事,还会回望自己怎么做、做得怎么样、哪里要改」的能力。

但**蜕变才刚开始**。下一步:**V0.0.6 self-improvement loop** 准备启动——让 LLM 从 `memory_corrections` 数据池自动总结萃取 prompt 改进点,实现「**人类反馈 → 自动学习 → 萃取质量持续提升**」的全自动数据飞轮;Tracing 数据反向喂回 Verifier Rubric,从「哪一维总是失败」自动调整权重;离线评测接入 CI,新功能合入前自动跑 baseline 防退化。当 Comet 不仅会回望自己,还能**从回望中自动学习、自动改进、不断蜕变**,它才真正配得上「蜕变」二字——**每一次回望,都是一次新生**。
